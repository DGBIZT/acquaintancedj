# Знакомство с Django
## Создание проекта интернет магазина

1. Описание:
```
 Cоздание проекта интернет-магазина, который будете дорабатывать на каждом уроке в течение всего курса.
```
2. Клонируйте репозиторий:
```
https://github.com/DGBIZT/acquaintancedj.git
```
###### №1
```
1. Настройка проекта.
2. Создание и настройка приложения
3. Создание шаблонов
    * home.html
    * contacts.html
4. Реализация контроллеров
```
###### №2
```
1. Подключение СУБД PostgreSQL
    * Создание БД
    * Внесение изменений в настройки подключения в файле setting.py
2. В приложении каталога созданы модели Product и Category и описаны для них базовые настройки.
      Product:
        * наименование,
        * описание,
        * изображение,
        * категория,
        * цена за покупку,
        * дата создания,
        * дата последнего изменения.
      Category:
        * наименование,
        * описание.
3. Перенесено отображение моделей в базу данных с помощью инструмента миграций. 
      Для этого:
        * созданы миграции для новых моделей;
        * применены миграции.
4. - Создание суперпользователя.
   - Зарегистрированы модели Product и Category в админке.
   - Настроены отображение для моделей:
        * Для Category выведены id и name в списке.
        * Для Product выведены id, name, price и category в списке.
        * Настроены фильтрации продуктов по категории.
        * Настроены поиск по полям name и description
5. Через инструмент shell заполнен список категорий, а также выберите список категорий, применив произвольные рассмотренные фильтры.
    1) Вход в Django shell
    2) Создано несколько категорий и продуктов.
    3) Выполнены следующие запросы:
        * Получите все категории.
        * Получите все продукты.
        * Найдены все продукты в определенной категории.
        * Обновление цены для определенного продукта.
        * Удалите продукта.
    4) Все решения сохранены в папке media/screenshots в виде скриншотов.
6. Сформированы фикстуры для моделей Category и Product
7. - Создана кастомная команда для добавления тестовых продуктов.
   - В команде прописано удаление всех существующих данных из базы перед добавлением новых продуктов.
```
###### №3
```
1. Создание нового контроллера и шаблона для отображения страницы с подробной информацией о товаре. 
   На этой странице показана вся информация о товаре.
2. Добавление в шаблон главной страницы код для отображения списка товаров с помощью цикла. 
   Чтобы карточки товаров выглядели одинаково, обрезано отображаемое описание до первых 100 символов.
3. Из-за увеличения количества шаблонов возникает много повторяющегося кода. Выделен общий (базовый) шаблон, 
   который будет включать общие элементы страницы, такие как шапка, подвал и стили. 
   Также создан подшаблон для главного меню, который можно будет включать в другие шаблоны.
```
###### №4
```
1. Переведены имеющиеся в проекте контроллеры с FBV на CBV.
2. - Создано новое приложение для блога и добавлено в файл settings.py. 
   - Создана новая модель блоговой записи со следующими полями:
        * заголовок,
        * содержимое,
        * превью (изображение),
        * дата создания,
        * признак публикации (булевое поле),
        * количество просмотров.
   - Для работы с блогом реализован полный CRUD для новой модели, используя CBV.
3. Модифицирован вывод и обработка запросов, добавив следующую логику на уровне контроллеров:
   - Увеличение счетчика просмотров: при открытии отдельной статьи увеличивать счетчик просмотров.
   - Фильтрация опубликованных статей: вывод в список статей только те, которые имеют положительный признак публикации.
   - Перенаправление после редактирования: после успешного редактирования записи необходимо перенаправлять пользователя на просмотр этой статьи.
```
###### №5
```
1. Реализован механизм CRUD для модели продуктов, используя модуль django.forms
   При этом обеспечена безопасность сайта от спама и не запрещено присваивать продуктам и добавлять
   в описание слова, которые включены в список запрещенных слов.
   Реализована валидация формы, что бы проверять отсутствие этих слов (в любом регистре) в данных полях
   Запрещенные слова
        * казино            * биржа             * обман
        * криптовалюта      * дешево            * полиция
        * крипта            * бесплатно         * радар
2. Добавлена кастомная валидация для поля purchase_price в форме создания и редактирования продуктов.
   Валидация проверяет, что цена продукта не может быть отрицательной.
3. Добавлена стилизация форм для продуктов, используя метод  __init__.
```
###### №6
```
1. Создана группа для роли "Модеатор продуктов" и настроены необходимые разрешения.
    - Добавлено к модели продукта поле, определяющее статус публикации. Это поле BooleanField со значением Falseпо умолчанию.
      Значение по умолчанию указывает на статус, не предполагающий публикацию продукта.
    - Реализована кастомная команда для создания группы
        * Добавлена кастомные права к модели продукта. can_unpublish_product — может отменять публикацию продукта.
        * Создана группа «Модератор продуктов» и назначены ей следующие права:
            - Разрешение can_unpublish_product.
            - Разрешение на удаление любого продукта.
        * Настроены проверки в контроллерах и шаблонах, чтобы использовать эти права.
          Проверены наличие прав у пользователя при попытке отменить публикацию или удалить продукт.

2. Реализовано решение, которое проверяет, что редактирование и удаление продукта доступно только его владельцу.
    - Добавлено поле владельца (owner) к модели продукта. Поле связано с моделью пользователя через ForeignKey
    - Настроено создание продукта таким образом, чтобы автоматически привязывать продукт к авторизованному пользователю.
      При создании нового продукта поле owner автоматически заполняться текущим авторизованным пользователем.
    - Настроены проверки в контроллерах, чтобы только владелец мог редактировать и удалять продукт. Реализована проверка 
      прав доступа в представлениях для редактирования и удаления продуктов, чтобы только владелец мог выполнять эти действия.
      Удалять продукты должен не только владелец, но и модератор.
    - Обновлены шаблоны, чтобы кнопка редактирования и удаления отображалась только для владельца продукта.
```
